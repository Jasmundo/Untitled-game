Detailed Description

Used to control the process of spawning dungeon floors.

Immediately after starting the game or getting to the next floor 9 versions of the dungeon get spawned. Only the "best" one stays. Logic behind determining scores and finalizing dungeons happens in the Update() function because the algorythm spawning a single dungeon version also exists within an Update() function.
